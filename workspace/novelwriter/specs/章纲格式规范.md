# 章纲格式规范 v2.0

所有章节的章纲（存放在 `## 本章概要` 中）必须严格遵循以下格式，AI 生成和手动编辑均需遵守。

## 模板

```markdown
### 章节信息
- 编号：[数字]
- 标题：[章节标题]
- 时间：[故事内时间]
- 地点：[主要场景]
- 出场人物：[人物1]、[人物2]、[人物3]
- 视角人物：[视角角色]

### 本章目标
[本章剧情推进的核心目的，一句话概括]

### 本章剧情概述
[2-4段叙述性文字，概述本章从开始到结束的全貌，包含核心情节走向]

### 剧情流程
1. [剧情点1：含场景/动作/结果]
2. [剧情点2：含场景/动作/结果]
3. [剧情点3：含场景/动作/结果]
4. [剧情点4：含场景/动作/结果]
5. [剧情点5：含场景/动作/结果]

### 本章冲突
- 外部冲突：[角色与外界的矛盾]
- 内部冲突：[角色内心的挣扎]
- 人际冲突：[角色之间的关系矛盾]

### 人物变化
- [角色A]：[变化描述]
- [角色B]：[变化描述]

### 释放信息
1. [通过本章剧情揭示的信息1]
2. [通过本章剧情揭示的信息2]

### 埋下伏笔
1. [为后续剧情埋下的伏笔1]
2. [为后续剧情埋下的伏笔2]
3. [为后续剧情埋下的伏笔3]

### 本章情绪基调
[本章的情感氛围，多个关键词逗号分隔]

### 本章看点/爽点/泪点
- 看点：[读者觉得精彩的部分]
- 爽点：[读者觉得爽的部分]
- 泪点：[读者觉得感动的部分]

### 章节结尾钩子
[章节结尾留下的悬念，一句话推动读者追下一章]

### 承接上一章
[上一章内容的简要回顾，说明本章如何衔接]

### 引出下一章
[下一章的内容预告，说明本章为下一章做了什么铺垫]

### 描写重点
[本章需要重点描写的场景、动作、心理等]

### 预计字数
本章正文约[N]字

### 备注
[额外需要注意的事项、写作建议等]
```

## 字段要求

| 字段 | 必填 | 说明 |
|------|------|------|
| 编号 | 是 | 阿拉伯数字 |
| 标题 | 是 | 章节标题 |
| 时间 | 是 | 故事内时间，如"第二天上午"、"三日后" |
| 地点 | 是 | 本章主场景地点 |
| 出场人物 | 是 | 本章出场角色，逗号分隔 |
| 视角人物 | 是 | 本章POV角色，只能一个 |
| 本章目标 | 是 | 一句话，本章在全书或当前剧情弧中的作用 |
| 本章剧情概述 | 是 | 2-4段，概述本章核心剧情 |
| 剧情流程 | 是 | 至少5条，按时间顺序，每条描述一个完整情节节点 |
| 本章冲突 | 是 | 外部/内部/人际各至少一句话，必须具体到本章内容 |
| 人物变化 | 是 | 至少一个角色，描述本章中的角色成长或变化 |
| 释放信息 | 是 | 至少1条 |
| 埋下伏笔 | 是 | 至少1条 |
| 本章情绪基调 | 是 | 关键词形式 |
| 本章看点/爽点/泪点 | 是 | 三项都必须填写，不得留空 |
| 章节结尾钩子 | 是 | 一句话，必须能吸引读者追下一章 |
| 承接上一章 | 是 | 不能省略 |
| 引出下一章 | 是 | 不能省略 |
| 描写重点 | 是 | 至少一条 |
| 预计字数 | 是 | 格式：本章正文约[N]字 |
| 备注 | 否 | 可省略 |

## 生成原则

1. 每个字段都必须填写（备注除外），不得留空或写"无"
2. 剧情流程至少5条，按时间顺序排列，每条描述一个完整情节节点
3. 冲突必须具体到本章内容，不能是泛泛的角色设定
4. 看点和钩子是网文核心，必须突出——没有看点的章节不如不写
5. 释放信息和埋下伏笔用于追踪剧情线，必须具体到本章揭示/埋设的内容
6. 章纲内容必须基于项目设定和正文生成，不得与已有设定冲突
7. AI 生成时必须使用上述完整模板，不得删减字段或添加模板之外的字段/标题
8. 人物变化要描述角色的内心转变，而非动作总结
